# kaoshi 管理端信息架构重建计划

## 需求

- 重建管理端中部门、题库、试题和考试管理的使用体验，让页面结构贴合真实考试平台的业务逻辑。
- 部门用树形组织管理，清楚表达上下级关系。
- 题库和试题作为绑定关系统一呈现：左侧题库树，右侧试题列表。
- 考试组卷同时支持按题库规则抽题和从试题明细中手工选题，最终都落到可排序、可预览、可发布的试卷题目明细。
- 优化用户导入模板的字典清单，让部门、角色、状态按可填写清单展开，而不是挤在一个单元格里。
- 保持 kaoshi 快速演进原则：需要重建页面、接口或数据结构时，直接建立当前主干，不保留历史兼容包袱。
- `start_dev.py` 已有未提交小修：开发启动时优先打开匿名浏览器；本计划执行时需要纳入验证和收口。

## 当前事实

- `spec.md` 确认当前 kaoshi 已实现用户、角色、部门、题库、试题、考试、发布快照、作答快照、评分和成绩主链路。
- 当前部门管理使用 `el-table` 的树表格，功能上支持父子部门、新建下级、编辑和删除，但视觉上层级感弱，不像目录树。
- 当前题库管理和试题管理是两个独立页面；试题可按题库筛选，但不是左侧题库树加右侧试题列表的一体化工作台。
- 当前题库只有分类字段，没有题库树工作台，也没有在题库节点上直接展示题量。
- 当前考试管理已支持按题库规则组卷、生成题目明细、题目明细排序、保存读回和发布快照。
- 当前考试管理还不支持在组卷时从试题列表直接手工选题。
- 当前用户导入模板和导出文件包含字典清单，但部门和角色以同一单元格内拼接文本展示，不利于人工填写。
- 当前试卷操作已有保存草稿、发布考试和关闭考试；复制试卷、下载试卷、撤销发布、禁用等操作尚未形成明确当前事实。
- 截图仅作为成熟平台交互参考，不要求 kaoshi 外观和截图完全一致。

## 生产级验收保护点

- 部门树必须能清楚表达父子层级、展开收起、选中态和当前部门详情。
- 部门新增下级、编辑、删除后必须真实保存，并刷新读回。
- 题库树必须展示分类、题库和题量，点击题库后右侧试题列表只显示该题库试题。
- 试题新增、编辑、导入后，题库树题量和右侧列表必须同步刷新。
- 试题仍然必须归属于一个题库；没有题库不能创建试题。
- 考试组卷必须把规则抽题和手工选题统一落到题目明细；题目明细保存后必须读回，发布必须按明细生成快照。
- 发布快照和已提交成绩不能被题库、试题或试卷后续编辑破坏。
- 用户导入模板必须让人工知道部门、角色、状态可以填什么，并能用模板真实导入。
- 真实浏览器验收必须覆盖部门树、题库树、试题列表联动、题量刷新、用户模板、规则组卷、手工选题、试卷预览和发布。

## 目标

- 把部门管理重建为左侧部门树、右侧部门详情和编辑操作区。
- 把题库管理和试题管理重建为一体化题库试题工作台；保留现有路由入口时也指向同一当前主干体验。
- 给题库节点增加题量统计，并在试题创建、编辑、导入后刷新。
- 在考试管理中增加手工选题能力，从题库树和试题列表选择具体试题加入题目明细。
- 增加试卷预览，让发布前能查看题目、选项、答案、解析、附件、分值和顺序。
- 明确试卷操作边界：复制试卷和下载试卷作为当前增强目标；关闭考试保留；撤销发布只在没有作答记录时允许；已有人作答后不允许破坏发布快照。
- 优化用户 Excel 模板和导出字典清单。
- 同步 `spec.md`，只写实现并验证后的当前事实。
- 补齐后端测试、前端单元测试和真实浏览器测试。

## 不做范围

- 不照抄截图外观、品牌、颜色和布局细节。
- 不做旧页面兼容保留；如果页面主干需要重建，直接以当前主干替换。
- 不做旧数据库迁移兼容；数据库脚本表达当前初始化事实。
- 不新增主观题人工阅卷、填空题、判断题、排序题、阅读理解等新题型。
- 不做复杂拖拽库依赖；题目明细排序优先保持当前上移、下移和序号调整，除非现有组件能低成本承载。
- 不让已发布且已有作答的考试被撤销或改写历史快照。

## 设计

- 部门管理：
  - 左侧使用 Element Plus 树形组件承载组织结构。
  - 节点展示部门名称、状态和下级数量。
  - 节点操作支持新建下级、编辑、删除。
  - 右侧展示当前选中部门详情、直属下级和可编辑表单。

- 题库试题工作台：
  - 左侧树包含分类和题库节点，题库节点显示题量。
  - 右侧展示当前题库下的试题列表、筛选、新建、编辑、导入和导出。
  - 新建试题默认归属当前选中题库。
  - 题库节点和试题列表共用同一事实源，保存后读回。

- 用户导入模板：
  - `用户导入` sheet 保留账号、姓名、部门、角色、状态。
  - `字典清单` sheet 改成多列表或多区块展开：部门一行一个，角色一行一个，状态一行一个。
  - 导入规则保持以部门名称、角色名称或角色编码匹配。

- 考试组卷：
  - 保留按题库规则组卷。
  - 新增手工选题入口，从题库树和试题列表选择题目加入题目明细。
  - 题目明细作为唯一发布来源，规则抽题和手工选题都写入题目明细。
  - 题目明细支持修改分值、排序、删除和预览。

- 试卷操作：
  - 复制试卷：复制考试配置、开放范围、组卷规则和题目明细，生成新草稿。
  - 下载试卷：导出当前发布快照或草稿题目明细，包含题目、选项、答案、解析和分值。
  - 关闭考试：保留当前能力。
  - 撤销发布：只允许没有 attempt 或 result 的考试回到草稿；已有作答则拒绝。
